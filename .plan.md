# v0.9.0 — Baseline & Diff Scanning

## Goal
Enable CI/CD-friendly scanning by comparing current results against a baseline.
Only new (unseen) findings are reported. This prevents alert fatigue in mature
codebases while catching newly introduced issues.

## Files to create/modify

### New files
- `internal/baseline/baseline.go` — core baseline logic (load, save, diff)
- `internal/baseline/baseline_test.go` — tests
- `cmd/raven/testdata/` — sample baseline for integration tests

### Modified files
- `internal/engine/models.go` — add NewFindings, BaselineFindings fields to Result
- `internal/cli/scan.go` — add `--baseline` and `--update-baseline` flags
- `internal/output/formatter.go` — show diff stats in pretty/json/sarif output
- `internal/engine/engine.go` — integrate baseline filtering into Scan()

## Architecture

```
Scan() → [findings] → Baseline.Diff(findings) → Result{
    Findings: all findings (for SARIF completeness)
    NewFindings: findings not in baseline
    BaselineFindings: findings that were in baseline
}
```

### Baseline format (JSON)
```json
{
  "version": "1.0",
  "generated_at": "2026-04-22T22:00:00Z",
  "findings": [
    {
      "rule_id": "js-sqli-001",
      "file": "src/api.js",
      "line": 42,
      "column": 12,
      "snippet_hash": "sha256:abc..."
    }
  ]
}
```

Matching logic:
1. Exact match on (rule_id, file, line) → same finding
2. If line moved: match on (rule_id, file, snippet_hash) with line tolerance ±5
3. Otherwise → new finding

### CLI flags
- `--baseline <file>` — path to baseline JSON
- `--update-baseline` — write current findings to `.raven-baseline.json`

### Exit codes
- `--baseline` set: exit 1 only if NewFindings > 0
- Normal mode unchanged

## Testing

- Integration: scan with baseline, verify only new findings reported

## Backwards compatibility
